01 — Introducción

⚡Como funciona

MikroTik envia eventos de su sistema de logging a un servidor remoto via UDP (protocolo syslog estándar). Syslog Alert corre como servicio de Windows, escucha esos paquetes, filtra por severidad, topic y palabras clave, y reenvía las alertas que te interesan al canal configurado.

Un solo servidor puede recibir alertas de múltiples routers simultáneamente. Cada mensaje incluye el identity del equipo. El sistema incluye un rate limiter anti-flood que suprime alertas repetidas y notifica al administrador cuando detecta un ataque sostenido.

[tuIdentity] [CRITICAL] login failure for user admin from 1.2.3.4 via winbox
02 — Antes de empezar

📋Requisitos

01

Windows 10 / 11

O Windows Server 2016 o superior. No requiere .NET instalado.

02

NSSM incluido

Incluido en el paquete. El instalador lo copia automáticamente.

03

Puerto UDP 514

Abierto en el firewall. El instalador lo configura automáticamente.

04

MikroTik RouterOS

Versión 6.x o 7.x. Compatible con cualquier modelo.

03 — Setup

🚀Instalacion

  1. Descomprimí el paquete en cualquier carpeta
  2. Ejecutá install.cmd como Administrador
  3. El instalador copia archivos, crea regla de firewall y registra el servicio
  4. Al finalizar se ofrece abrir la documentación y archivo de configuración
  5. Editá SyslogAlerter.cfg con tus datos de Telegram y/o WhatsApp
  6. Reiniciá el servicio para aplicar la configuración
Nota: El servicio queda corriendo en background y arranca automáticamente con Windows.
04 — Configuracion

⚙️SyslogAlerter.cfg

# --- General ---
Puerto=514

# --- Severidades a alertar (separadas por coma) ---
Severidades=warning,error,critical,alert,emergency

# --- Topics a ignorar (separados por coma) ---
TopicsIgnorar=bfd,ospf,rip,meter

# --- Palabras clave a ignorar en mensajes (separadas por coma) ---
# Si el mensaje contiene cualquiera de estas palabras, se filtra
MensajesIgnorar=possible SYN flooding,possible port scan,possible IP spoofing

# --- Telegram ---
# true = activado | false = desactivado
TelegramEnabled=false
TelegramToken=TU_BOT_TOKEN_AQUI
TelegramChatId=TU_CHAT_ID_AQUI

# --- WhatsApp (textmebot) ---
# true = activado | false = desactivado
WhatsAppEnabled=false
WhatsAppPhone=+549XXXXXXXXXX
WhatsAppApiKey=TU_API_KEY_AQUI

Severidades disponibles

DEBUG  INFO  NOTICE  WARNING  ERROR  CRITICAL  ALERT  EMERGENCY

Filtro de palabras clave (v2.2+)

Ignorá mensajes que contengan ciertas palabras sin enviar alertas. Caso insensible.

MensajesIgnorar=possible SYN flooding,possible port scan,brute force attempt

Como obtener el Token de Telegram

  1. Abrí Telegram y buscá @BotFather
  2. Enviá /newbot y seguí las instrucciones
  3. BotFather te devuelve el token — copialo en TelegramToken

Como obtener el ChatId de Telegram

  1. Buscá @userinfobot en Telegram
  2. Enviá cualquier mensaje
  3. Te responde con tu ChatId — copialo en TelegramChatId
05 — MikroTik

🔧Configuracion en MikroTik

Ejecutá los comandos en orden desde la terminal de MikroTik. Reemplazá los valores indicados.

1. Configurar el identity del equipo

/system identity set name=tuIdentity

⚠ Reemplazá tuIdentity por el nombre de tu equipo

2. Crear la logging action

/system/logging/action
add name=syslogAlert target=remote remote=IP.O.DOMINIO remote-port=514 remote-log-format=default

⚠ Reemplazá IP.O.DOMINIO por la IP del servidor

3. Configurar el prefix

/system/logging set [find action=syslogAlert] prefix=[/system identity get name]

4. Crear las logging rules

/system/logging
add topics=system,warning  action=syslogAlert
add topics=system,error    action=syslogAlert
add topics=firewall        action=syslogAlert

Bloque completo para copiar y pegar

Reemplazá tuIdentity e IP.O.DOMINIO antes de ejecutar.

/system identity set name=tuIdentity

/system/logging/action
add name=syslogAlert target=remote remote=IP.O.DOMINIO remote-port=514 remote-log-format=default

/system/logging set [find action=syslogAlert] prefix=[/system identity get name]

/system/logging
add topics=system,warning  action=syslogAlert
add topics=system,error    action=syslogAlert
add topics=firewall        action=syslogAlert

⚠ Personaliza antes de ejecutar

06 — Operacion

🛠Administrar el servicio

Desde PowerShell como Administrador:

# Ver estado
Get-Service SyslogAlert

# Detener
Stop-Service SyslogAlert

# Iniciar
Start-Service SyslogAlert

# Reiniciar (necesario después de editar .cfg)
Restart-Service SyslogAlert

🗑Desinstalar
