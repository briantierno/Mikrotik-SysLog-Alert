01 — Introducción

⚡Como funciona

MikroTik puede enviar eventos de su sistema de logging a un servidor remoto via UDP (protocolo syslog estándar). Syslog Alert corre como servicio de Windows, escucha esos paquetes, filtra por severidad y topic, y reenvía las alertas que te interesan al canal configurado.

Un solo servidor puede recibir alertas de múltiples routers simultáneamente. Cada mensaje incluye el identity del equipo que lo generó. El sistema incluye un rate limiter anti-flood que suprime alertas repetidas y notifica al administrador cuando detecta un ataque sostenido.

[tuIdentity] [CRITICAL] login failure for user admin from 1.2.3.4 via winbox
02 — Antes de empezar

📋Requisitos

01

Windows 10 / 11

O Windows Server 2016 o superior. No requiere .NET instalado.

02

NSSM incluido

Incluido en el paquete. El instalador lo copia automáticamente.

03

Puerto UDP 514

Abierto en el firewall de Windows. El instalador lo configura automáticamente.

04

MikroTik RouterOS

Versión 6.x o 7.x. Compatible con cualquier modelo.

03 — Setup

🚀Instalacion

  1. Descomprimí el paquete en cualquier carpeta
  2. Ejecutá install.cmd como Administrador
  3. El instalador copia los archivos, crea la regla de firewall y registra el servicio
  4. Al finalizar se ofrece abrir la documentación y el archivo de configuración
  5. Editá SyslogAlerter.cfg con tus datos de Telegram y/o WhatsApp
  6. Reiniciá el servicio para aplicar la configuración
Nota: El servicio queda corriendo en segundo plano y arranca automáticamente con Windows.
04 — Configuracion

⚙️SyslogAlerter.cfg

# --- General ---
Puerto=514

# --- Severidades a alertar (separadas por coma) ---
# Opciones: debug, info, notice, warning, error, critical, alert, emergency
Severidades=warning,error,critical,alert,emergency

# --- Topics a ignorar (separados por coma) ---
TopicsIgnorar=bfd,ospf,rip,meter

# --- Telegram ---
# true = activado | false = desactivado
TelegramEnabled=true
TelegramToken=TOKEN_DEL_BOT
TelegramChatId=ID_DEL_CHAT

# --- WhatsApp (textmebot) ---
# true = activado | false = desactivado
WhatsAppEnabled=false
WhatsAppPhone=+549XXXXXXXXXX
WhatsAppApiKey=API_KEY

Severidades disponibles

DEBUG  INFO  NOTICE  WARNING  ERROR  CRITICAL  ALERT  EMERGENCY

Como obtener el Token de Telegram

  1. Abrí Telegram y buscá @BotFather
  2. Enviá /newbot y seguí las instrucciones
  3. BotFather te devuelve el token — copialo en TelegramToken

Como obtener el ChatId de Telegram

  1. Buscá @userinfobot en Telegram
  2. Enviá cualquier mensaje
  3. Te responde con tu ChatId — copialo en TelegramChatId

Como obtener la ApiKey de WhatsApp (textmebot)

  1. Entrá a https://www.textmebot.com
  2. Registrate con tu número de WhatsApp
  3. Te asignan una ApiKey — copiala en WhatsAppApiKey
05 — MikroTik

🔧Configuracion en MikroTik

Ejecutá los siguientes comandos en orden desde la terminal de MikroTik. Reemplazá los valores indicados antes de ejecutarlos.

1. Configurar el identity del equipo

El identity es el nombre que aparecerá en cada alerta para identificar el equipo. Configuralo antes de continuar.

/system identity set name=tuIdentity

⚠ Reemplazá tuIdentity por el nombre descriptivo de tu equipo (ej: Router-Oficina, MikroTik-Sucursal)

2. Crear la logging action

Reemplazá IP.O.DOMINIO por la IP pública o dominio DDNS del servidor Windows.

/system/logging/action
add name=syslogAlert target=remote remote=IP.O.DOMINIO remote-port=514 remote-log-format=default

⚠ Reemplazá IP.O.DOMINIO por la IP o dominio de tu servidor

3. Configurar el prefix con el identity

El prefix incluye automáticamente el identity del equipo en cada mensaje enviado al servidor.

/system/logging set [find action=syslogAlert] prefix=[/system identity get name]

4. Crear las logging rules

/system/logging
add topics=system,warning  action=syslogAlert
add topics=system,error    action=syslogAlert
add topics=firewall        action=syslogAlert

Configuración completa en un solo bloque

Copiá el bloque completo y pegalo en la terminal de MikroTik. Asegurate de reemplazar tuIdentity e IP.O.DOMINIO antes de ejecutarlo.

/system identity set name=tuIdentity

/system/logging/action
add name=syslogAlert target=remote remote=IP.O.DOMINIO remote-port=514 remote-log-format=default

/system/logging set [find action=syslogAlert] prefix=[/system identity get name]

/system/logging
add topics=system,warning  action=syslogAlert
add topics=system,error    action=syslogAlert
add topics=firewall        action=syslogAlert

⚠ Reemplazá tuIdentity e IP.O.DOMINIO antes de ejecutar

5. Verificar que funciona

Intentá conectarte al router con usuario y contraseña incorrectos via Winbox o SSH desde otra PC. Deberías recibir una alerta como esta:

[tuIdentity] [CRITICAL] login failure for user admin from 1.2.3.4 via winbox
06 — Operacion

🛠Administrar el servicio

Desde PowerShell como Administrador:

# Ver estado
Get-Service SyslogAlert

# Detener
Stop-Service SyslogAlert

# Iniciar
Start-Service SyslogAlert

# Reiniciar (necesario despues de editar el .cfg)
Restart-Service SyslogAlert

Cuando reiniciar el servicio

  • Despues de editar el archivo SyslogAlerter.cfg
  • Si las alertas dejan de llegar
  • Despues de actualizar el ejecutable

Agregar otro MikroTik

No requiere ningún cambio en el servidor. Simplemente configurá la logging action en el nuevo router apuntando al mismo servidor. Las alertas llegan identificadas con el identity de cada equipo.

07 — Desinstalacion

🗑Desinstalar

Ejecutá uninstall.cmd como Administrador. El desinstalador:

  1. Detiene el servicio de Windows
  2. Elimina el servicio de Windows
  3. Elimina la regla de firewall UDP 514
  4. Elimina los archivos de C:\Program Files\SyslogAlert

Formato syslog: El formato recomendado en MikroTik es default. El formato bsd-syslog también es compatible, pero con default el parseo es más preciso y confiable. Usá bsd-syslog solo si necesitás ver el mensaje RAW completo para debug.
